\section{Limitations and Reproducibility}

The paper makes three kinds of claim, and each one breaks differently. That is
the main limitation to manage.

For first-pass Gemma pruning, the evidence is now good enough for a main
results section, but not every cell is equally clean. VideoMME is clean.
MVBench and TOMATO are advisory because the measured gains survive, but the
thermal caveats are still visible and should remain visible in the text.
Architecture breadth is still asymmetric, but no longer Gemma-only. The
first-pass measured headline cells are still Gemma-first today, while a
matched Qwen cross-architecture probe now shows the same ceiling binding at a
smaller absolute lift. The pooled n=60 H\_stack result also closes one
tempting overclaim: composition does not outrun the ceiling. The local stacked
cell is ceiling-matched, which is useful evidence for the arithmetic and a
null on super-ceiling composition.

For persistent-KV, the numbers are much larger, but the regime is narrower.
These are \emph{after-ingest} claims: the first query on the video still pays
full freight, and the gain only appears when subsequent questions concern that
same video. The safe envelope is also finite. Through 16 frames the local curve
stays inside the current 7B safe envelope, with a clean 16f point and a
smaller-but-still-safe 8f point. Beyond that, unrepaired fidelity degrades
quickly. The 3B story is wider but weaker: it holds a tolerated $-0.19$
pre-basin plateau through 36f / 14.5k prefill tokens rather than preserving the
clean 7B envelope, and the current mechanism story is threshold onset plus
basin geometry rather than an architecture-specific saturation ceiling.

Selective re-prefill narrows that limitation but does not erase it. The K=1
20f Qwen 7B result is now a real local repair frontier, with no observed paired
drift on 51 short+medium paired queries and zero observed pathological
follow-up outputs across 34 repaired follow-ups. It is still not a universal
persistent-KV fix: long-bucket and 32f boundary probes remain open, and the
medium run exceeded its too-tight preregistered RSS ceiling while still staying
inside the local 9\,GB guard. The failed \(Q2=K1,Q3=K0\) adaptive variant also
matters: it shows that skipping the third-query repair by falling back to the
retained full cache reintroduces the basin. A better adaptive state path is
still untested because the first \(Q3\)-from-repaired-state attempt exposed a
runner bug on a text-only tail.

For Qwen routing, the opposite caveat applies. The holdout evidence is clean
and useful, but it is still a quality--compute frontier result under a fixed
dense backend. It should not be inflated into a measured sparse-execution
speedup claim until the backend actually skips timed work. The same caution now
applies to the Qwen VideoMME frame-count story: the dev-split 16f long-bucket
regression does not replicate on disjoint holdout, so the manuscript treats it
as a dev-only shape rather than a universal scaling law. The 1.57 feature-drift
geometry itself now reproduces on holdout at 8f and 16f; what failed to
transfer was the downstream long-bucket accuracy shape, not the drift signal.

The streaming lane is the smallest-$N$ evidence class in the paper. It is
there because it shows the practical regime where redundancy becomes large, not
because it already closes the full systems-reviewer checklist. The paper still
needs a screenshot-polling baseline and broader UI coverage if that lane is
going to carry more of the headline later. The local Qwen session/streaming
bridge now exists and is useful precisely because it is diagnosed rather than
promoted: the original stack reproduced speedup but lost accuracy, dense-Q0
admission fixes Q0 exactly but misses the 3-query rescue gate, and the offline
admission frontier contains an oracle rescue point but no deployable
format-clean policy in the current endpoint family. The 1.30Z continuation
then falsifies the \(kr_{Q0}=0.67\) long-bucket candidate
(\(\Delta\)acc \(=-0.130\), 3.12$\times$, format-clean) and empirically
confirms that follow-up vision pruning is inactive under prompt-cache reuse
in that arm. The completed root-cause and continuation runs localize the
remaining bridge to Q0 admission policy or a deliberately cache-invalidated
follow-up V-only path, not to the current session-reuse path. A related
local codec-native bridge remains scientifically interesting, and the latest
1.29 results now provide semantic
planner-substitution evidence: MAX-over-span sparse sampling is hard-falsified,
but the continuous-score redesign matches dense choices on VideoMME dev
all-duration \(n=30\) and later calibration-mode/source ablations are neutral
on the local slices we ran. This remains a semantic planner result, not a
latency win; the offline extraction path took 7290 s and the deployment path
still needs streaming decoder integration.

The current benchmark mix also under-samples scroll and pan regimes. We audited
this directly on a 60-item VideoMME stratification across 8f/16f/32f and found
0/60 items above a relaxed \texttt{shifted\_fraction} \(\geq 0.30\) gate, with
max shifted fraction 0.125. That does not show C-VISION is safe on sustained
egomotion or scrolling content; it shows VideoMME is the wrong corpus for that
question. A scroll/pan characterization now requires EgoSchema, EPIC-Kitchens,
Ego4D, or a labeled synthetic set.

Finally, equal aggregate accuracy is not the same as identical behavior. The
Gemma routing split makes this visible: TOMATO passes the agreement gate, while
MVBench preserves aggregate accuracy but changes 8/30 choices across four
motion subgroups. The 1.30 and 1.55 drift audits show the same pattern in
different forms. Reviewers should read the paired-drift columns as part of the
claim, not as decorative diagnostics.

The paper is nevertheless being built to be reproducible. Automated main-text
figures and tables are generated from checked-in artifact summaries or from
checked-in paper figures, the appendix records the repo commits used for the
sync, and imported or advisory evidence is labeled as such instead of being
silently promoted into the headline figures. The paper builds locally with the
checked-in \texttt{make paper-*} targets plus `tectonic` or a fuller TeX
stack.
